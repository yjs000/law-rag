# 품질 점수표

평가일: 2026-07-13  
단계: 설계 스켈레톤

등급: A(출시 준비), B(관리되는 격차), C(핵심 미완성), D(미구현/고위험), N/A(해당 없음)

| 영역 | 등급 | 근거 | 다음 종료 조건 |
|---|---:|---|---|
| 제품 범위 | D | 사용자·자료 범위가 가정 상태 | 제품 명세 승인 |
| 아키텍처 | C | 경계 초안 존재, 기술 결정 없음 | 핵심 ADR 승인 및 구조 테스트 |
| 데이터 수집 | D | 제공처·이용조건 미검증 | 한 출처 종단 수집 검증 |
| 검색 품질 | D | 구현·평가셋 없음 | Recall 기준선 측정 |
| 답변/인용 | D | 계약 초안만 존재 | 인용 검증 종단 테스트 |
| 보안/개인정보 | C | 원칙과 위협 목록 초안 | 위협 모델 승인 및 통제 구현 |
| 신뢰성/운영 | D | SLO와 관측 구현 없음 | 핵심 SLI 계측 및 런북 |
| 프론트엔드/접근성 | D | 원칙만 존재 | 핵심 흐름 접근성 테스트 |
| 문서/에이전트성 | B | 색인·계획·가이드 존재 | 링크/신선도 CI 검사 |

## 갱신 규칙

- 중요한 릴리스 전과 월 1회 갱신한다.
- 등급에는 테스트, 대시보드, 평가 실행 등 검증 가능한 근거를 연결한다.
- 점수를 높이는 것보다 격차와 다음 종료 조건을 정확히 쓰는 것을 우선한다.
