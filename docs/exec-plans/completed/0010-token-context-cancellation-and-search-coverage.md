# 토큰 컨텍스트·서버 취소·검색 범위 개선

기준 시점: 2026-07-18  
상태: 완료

## 목적과 사용자 결과

- 메시지 개수가 아니라 모델 입력 토큰을 기준으로 대화 컨텍스트를 제한한다.
- 중지 버튼이 브라우저 요청뿐 아니라 진행 중인 서버 검색과 모델 호출을 취소한다.
- 조문 경로만 입력하거나 잘못된 법령명을 입력한 검색 전용 요청도 무응답 대신 검증 가능한 결과 또는 명확한 무결과 안내를 반환한다.
- Qwen3:4b 연결에 필요한 설정·보안·검증 항목을 확정한다.

## 범위와 비범위

- 범위: Web 컨텍스트 선택/전환, API 요청 컨텍스트, 요청 취소 API, 검색 전용 파서·무결과 처리, 테스트와 운영 문서.
- 비범위: Qwen 런타임 설치와 실제 운영 endpoint 배포, 임베딩 모델 교체·재색인.

## 완료 조건

- [x] 400개 메시지 기준이 런타임 판단과 UI에서 제거된다.
- [x] 32,768 native context 기준으로 입력 24,576, 출력 4,096, 시스템·근거·구조화 여유 4,096 토큰 정책을 문서화한다.
- [x] 최근 완료 턴만 예산 내에서 모델 입력에 포함하고, 초과 시 새 대화와 초기화 안내를 표시한다.
- [x] 요청 ID 기반 취소 endpoint가 실행 중 검색/모델 coroutine에 취소를 전파한다.
- [x] 조문 경로, 법령명 오인식, 하위 경로 부재, 시행일 경계를 포함한 검색 테스트가 통과한다.
- [x] Core/API/Web 포맷·린트·타입·테스트가 통과한다.

## 단계와 담당

- [x] 컨텍스트 agent: 토큰 추정·최근 턴 선택·경계 단위 테스트.
- [x] 취소 agent: 실행 태스크 레지스트리·취소 endpoint·API 테스트.
- [x] 검색 agent: 검색 범위 및 안전한 무결과 처리·테스트 행렬.
- [x] root: Web/API 컨텍스트 전달 통합, UI 취소 연동, Qwen 준비 문서, 전체 회귀.

## 검증 및 롤백

- Web Vitest, TypeScript, ESLint와 Core/API pytest를 실행한다.
- 취소 테스트는 검색 대기와 모델 대기 각각에서 실제 coroutine 취소를 확인한다.
- 각 기능은 독립 커밋으로 남겨 문제가 생기면 해당 커밋만 되돌릴 수 있게 한다.

## 결정 로그

- 2026-07-18: 메시지 수는 문자열·근거 크기를 반영하지 못하므로 컨텍스트 제한 지표로 사용하지 않는다.
- 2026-07-18: Qwen3-4B의 native 32K 범위를 기본으로 사용하고 장문 확장(YaRN)은 운영 평가 전 활성화하지 않는다.
- 2026-07-18: 클라이언트 연결 종료만으로 취소 완료를 가정하지 않고 서버가 요청 태스크를 명시적으로 관리한다.

## 진행 기록

- 2026-07-18: 요구사항 재확인, 병렬 담당 분리, 토큰 추정 및 경계 단위 구현 완료.
- 2026-07-18: Web 컨텍스트 전달과 취소 UI를 API에 연결하고, 검색·생성 취소 및 검색 경계 회귀를 검증했다.
- 2026-07-18: Core 3, API 172, Web 46 테스트와 Ruff, TypeScript, ESLint, production build가 통과했다.

## 결과와 잔여 작업

- 400 메시지 기준은 폐기되었고 실제 모델 입력용 최근 완료 턴만 토큰 예산 내에서 전달한다.
- 검색/생성 coroutine 취소는 단일 프로세스에서 검증됐다. 다중 인스턴스 분산 취소는 운영 배포 전 Redis/PostgreSQL 신호 또는 sticky routing으로 확장해야 한다.
- Qwen 실제 연결 후 tokenizer 오차와 runtime `num_ctx`를 smoke test해야 한다.

## 미결정과 차단 요소

- 실제 Qwen endpoint 위치와 인증 방식은 사용자 배포 환경 결정이 필요하다.
- 실제 tokenizer를 연결한 뒤 보수 추정치와의 오차율을 고정 평가셋에서 재보정해야 한다.
