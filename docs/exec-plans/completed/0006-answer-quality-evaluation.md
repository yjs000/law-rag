# 실행 계획 0006: 예시 질문 기반 답변 품질 평가

상태: 완료
작성일: 2026-07-15
소유자: Codex 에이전트 팀

## 목적과 사용자 결과

대표 에너지 법령 질문마다 어떤 문서·조문을 근거로 어떤 범위의 답변이 나와야 하는지 명시하고, 목업 코퍼스와 결정적 평가로 검색·답변·인용 회귀를 막는다. 사용자는 질문과 맞는 근거 후보, 근거 범위 안의 결론, 실행 가능한 확인 항목과 명시적 한계를 받는다.

## 범위와 비범위

범위는 허가, 변경·신고, 전기저장시설 안전, 직접 조문 경로, 근거 없음 예시의 기대 근거·필수 용어·금지 결론, 목업 검색 순위, 검색 전용 답변, 생성 초안 인용 게이트와 평가 스크립트다.

실제 사용자 로그인, 실제 사건 판단, 판례·외부 법률 사이트, 모델 기억으로 근거 보완은 제외한다. 실제 법령 정답 조문의 사람 검토나 최신 원문 추가 수집이 필요하면 즉시 보고하고 후순위 운영 과제로 남긴다.

## 측정 가능한 완료 조건

- 최소 5개 대표 질문에 기대 문서·조문 경로·필수 답변 용어·금지 과잉결론이 있다.
- 목업 코퍼스 검색에서 기대 근거가 Top 10에 있고 직접 조문 질의는 해당 경로를 우선한다.
- 검색 전용 응답은 질문과 관련된 근거 요약·체크리스트·정확 원문 인용을 제공한다.
- 생성 초안은 존재하지 않는 인용, 근거에 없는 규범어·숫자·과잉결론을 통과하지 못한다.
- 근거 없음은 결론을 생성하지 않고 질문 구체화와 자료 범위를 안내한다.
- 전체 테스트, lint, typecheck, 문서 검사와 production build가 통과하고 원격 브랜치에 push된다.

## 검증 및 롤백

외부 API 없이 고정 fixture와 목업 SearchHit으로 먼저 검증한다. 대표 명령은 `powershell -ExecutionPolicy Bypass -File scripts/verify.ps1`이며 평가 전용 명령을 추가하면 같은 검증 계약에 포함한다. 품질 회귀 시 코드 변경을 되돌리되 기존 검색 전용 안전 폴백은 유지한다.

## 실제 결과와 남은 부채

외부 API나 사용자 데이터 없이 대표 질문의 최소 답변 계약과 안전 하한선을 자동화했다. 검색 전용 응답은 근거별 제목·조문·원문·확인 항목을 제공하고, 생성 초안은 근거에 없는 의무·면제·숫자·자동 승인 같은 결론을 거부한다.

실제 서비스 코퍼스의 Recall@k·MRR/nDCG, 실제 Terra 답변의 정확성, 복합·상충 근거, 법률 전문가 블라인드 검토는 아직 증명되지 않았다. 이 항목들은 최신 국가법령정보 Open API 스냅샷과 사람 검토가 확보되는 운영 평가에서 수행한다.

## 미결정과 차단 요소

- 실제 서비스 코퍼스의 조문별 법률 해석 정답은 국가법령정보 Open API 최신 원문과 사람 검토가 필요하다. 이번 계획은 자동화 가능한 하한선과 과잉결론 방지를 먼저 확정한다.
