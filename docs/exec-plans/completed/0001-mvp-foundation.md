- [x] 문서 단위 정규화·정확 명칭·버전 검증이 모두 성공한 변경만 활성 manifest에 즉시 반영
- [x] manifest를 임시 파일 작성 후 원자 교체하고 중단·실패 시 직전 정상 상태를 보존
- [x] 가지조문, 부칙, 시행예정, 폐지·삭제 표식, 누락 필드 fixture와 경계 테스트 추가
- [x] 주 1회 스케줄과 즉시 반영·실패 격리·재실행 멱등성 운영 문서 및 학습 노트 갱신

### Agent C — 인용 의미 게이트와 API 보안

- [x] 답변의 각 주장·체크리스트가 가리키는 인용 원문과 핵심 용어가 일치하는 결정적 검증 게이트 구현
- [x] 근거가 비어 있거나 무관하거나 존재하지 않는 인용이면 AI 답변을 차단하고 검색 전용으로 폴백
- [x] 고정 평가기에 인용 존재율·인용 원문 일치율을 추가하고 실패 시 비정상 종료
- [x] 프롬프트 주입, 과도한 입력, 인증 우회, 허용되지 않은 원문 URL/SSRF, 로그 비밀 노출 회귀 테스트 추가
- [x] 질문 원문·원문 전문 없이 요청 ID·모드·결과만 남기는 구조화 관측 이벤트 경계 구현

### Agent D — 워크벤치 근거 탐색과 접근성

- [x] 답변/원문 좌우 패널과 선택 인용 강조·키보드 포커스 이동을 완성
- [x] 기준일·법률/시행령/시행규칙/행정규칙 필터와 코퍼스 최신성·누락 경고 표시
- [x] 익명 첫 질문 팝업의 초기 포커스·Tab 순환·Shift+Tab·ESC·포커스 복귀 테스트
- [x] 목업 로그인→질문→이력→삭제→Markdown/CSV/PDF 내보내기 상태 흐름 테스트
- [x] 원문과 모델 문자열을 HTML로 해석하지 않는 XSS 회귀 테스트 및 웹 학습 노트 갱신

### Root — 통합, 위협 모델, 완료 판정

- [x] 모든 MVP 완료 조건 충족 시 이 계획을 `completed/`로 이동

## 완료 결과

- Agent B/C/D 병렬 품질 강화와 루트 통합을 완료했다.
- 등록 IP 실호출 산출물은 MVP 9개 문서, 235개 버전이며 고정 평가 9/9, Recall@10 1.0을 유지했다.
- 인용 ID 존재율과 표시 인용-검색 원문 완전 일치율은 모두 100%였다.
- API·공용 코어 54개, collector 29개, web 11개 테스트와 린트·타입 검사·프로덕션 빌드가 통과했다.
- 공식 삭제 목록 `delHst`의 법령·행정규칙 동시 조회, JSON 우선/XML 폴백, 체크포인트 중첩 수집, 실패 격리와 원자 출처 상태 반영을 fixture 계약으로 검증했다. `delHst`는 법적 폐지와 분리하며 효력 종료일을 추론하지 않는다.
- 실제 브라우저에서 익명 질문, 검색 전용 답변·인용, 로그인 유도, ESC 포커스 복귀, 목업 로그인과 질문 이력을 확인했다.
- 잔여 실제 Supabase·Google OAuth·OpenAI·클라우드 연결은 [실행 계획 0002](../active/0002-production-connections.md)로 분리했다.
