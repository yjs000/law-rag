# 실행 계획 0003: 채팅 중심 웹 경험

상태: 완료
작성일: 2026-07-14
소유자: 미정

## 목적과 사용자 결과

기존 3열 법령 조사 워크벤치를 어두운 테마 우선의 채팅 화면으로 재구성한다. 익명 사용자는 저장 없이 질문과 답변을 이어갈 수 있고, 로그인 사용자는 왼쪽 대화 목록에서 저장된 질문을 다시 연다. 사용자는 답변 생성 전에 `gpt-5.6-terra` 또는 검색 전용 모드를 선택하고, 계정 화면에서 로그인 상태·질문 보존 기간·허용 모델과 제한을 확인한다.

## 범위와 비범위

범위에는 반응형 채팅 셸, 첫 화면 안내, 모델 선택, 질문 작성기, 인용·원문 표시, Google 목업 로그인, 대화 이력 열기·삭제, 계정 대시보드, 요청별 검색 전용 선택, 관련 제품·프론트·학습 문서 및 테스트가 포함된다.

다음은 이번 계획의 비범위다.

- Google OAuth와 Supabase의 실제 운영 연결
- `gpt-5.6-terra` 외 생성 모델 지원
- 익명 질문의 서버 저장 또는 로그인 뒤 소급 저장
- 계정별 유료 요금제·실사용량 과금

## 완료 조건

- 데스크톱은 왼쪽 대화 목록과 중앙 채팅, 모바일은 닫을 수 있는 사이드바로 동작한다.
- 익명 질문은 채팅으로 답변되지만 이력 API에 저장되지 않으며, 첫 답변 뒤 로그인 안내는 대화를 가리지 않는다.
- 로그인 사용자는 저장된 질문을 왼쪽 목록에서 열고 삭제할 수 있다.
- Terra와 검색 전용 선택이 API 요청에 전달되고 검색 전용 선택은 생성 모델을 호출하지 않는다.
- 다른 생성 모델은 지원 대상으로 오해되지 않도록 미지원으로 표시한다.
- 계정 화면에 계정 정보, 1년 보존, Terra 단일 모델, 검색 전용 폴백, 계정 한도 미결정 상태가 표시된다.
- 웹 lint·typecheck·test·build와 관련 API 단위·통합 테스트가 통과한다.
- 제품 명세, 프론트 문서, 설계 결정 기록, `docs/learning/` 기술 브리핑이 구현과 일치한다.

## 검증과 롤백

대표 검증 명령은 `pnpm lint:web`, `pnpm typecheck`, `pnpm test:web`, `pnpm build:web`, `uv run --project apps/api pytest apps/api/tests`이다. API 계약은 새 필드에 기본값을 두어 기존 클라이언트를 유지한다. 문제 발생 시 새 채팅 셸과 요청 모드 필드를 각각 독립적으로 되돌릴 수 있으며 데이터 마이그레이션은 없다.
