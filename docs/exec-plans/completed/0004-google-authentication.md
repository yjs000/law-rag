## 목적과 사용자 결과

현재 완성된 로그인·회원가입 UI를 실제 Supabase Google OAuth와 연결한다. 사용자는 Google 계정으로 가입하거나 로그인하고, 안전하게 세션을 복원하며, 자신의 질문 이력만 열고, 로그아웃하거나 계정 및 연결 데이터를 삭제할 수 있어야 한다.

이메일·비밀번호 가입, 비밀번호 재설정, Google 외 공급자, 조직·팀 계정은 비범위다.

## 완료 조건

- Production에서 Google 가입·로그인·로그아웃·세션 복원이 종단 동작하고 계정 삭제 경계는 가짜 ID 통합 테스트로 검증된다.
- 로그인 사용자는 자신의 질문 이력만 볼 수 있고 익명 질문은 저장되지 않는다.
- 서비스 role/secret key와 원문 토큰이 브라우저·로그·분석 이벤트에 노출되지 않는다.
- 동의하지 않은 가입, 변조 callback, 만료 토큰, 다른 사용자 이력 접근이 차단된다.

## 검증과 롤백

FastAPI 인증·RLS 통합 테스트, Next.js 인증 상태 테스트, staging Google OAuth 종단 테스트를 실행한다. 장애 시 실제 인증 진입을 닫고 익명 검색·질문은 유지한다. 목업 인증은 production에서 계속 비활성 상태여야 한다.
