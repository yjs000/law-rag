# 실행 계획 0001: MVP 기반 확정

상태: 활성  
작성일: 2026-07-13  
소유자: 미지정

## 목적

법률 RAG MVP의 사용자, 데이터 범위, 기술 스택, 평가 기준을 확정하고 한 개의 대표 질문을 수집부터 인용 답변까지 종단으로 통과시킨다.

## 완료 조건

- 우선 사용자와 핵심 사용자 여정 승인
- 데이터 제공처의 접근·이용·갱신 조건 기록
- 구현 언어, 저장소, 검색, 모델, 배포 방식 ADR 승인
- 최소 스키마와 마이그레이션 도구 도입
- 작은 법률 자료 집합의 수집/정규화/색인 성공
- 대표 질의의 검색 결과와 검증된 인용 답변 성공
- 고정 평가셋과 초기 기준값 생성
- 로컬 한 명령 실행, 테스트 및 CI 경로 문서화

## 단계

### 1. 제품 범위 결정

- [ ] 초기 사용자와 해결할 업무 확인
- [ ] 지원 관할, 문서 유형, 기준일 결정
- [ ] 비목표와 법률 자문 고지 승인

### 2. 데이터 적법성과 품질 확인

- [ ] 후보 제공처, API/다운로드 방법, 이용 조건 조사
- [ ] 샘플 원문으로 위치 식별자와 버전 모델 검증
- [ ] 갱신 주기와 삭제/정정 흐름 결정

### 3. 기술 결정

- [ ] 런타임·패키지 관리·웹 프레임워크 ADR
- [ ] 관계형/객체/검색 저장소 ADR
- [ ] 임베딩·재순위·생성 모델 및 추상화 경계 ADR
- [ ] 로컬 개발·CI·배포 환경 ADR

### 4. 수직 슬라이스 구현

- [ ] 스키마와 마이그레이션
- [ ] 한 데이터 제공처 수집/정규화
- [ ] 구조 보존 청킹과 색인
- [ ] 하이브리드 검색과 구조화 답변
- [ ] 인용 검증과 실패 응답
- [ ] 최소 UI 또는 API

### 5. 검증과 운영 준비

- [ ] 단위·통합·종단 테스트
- [ ] 평가셋 및 회귀 명령
- [ ] 구조화 로그, 메트릭, 추적
- [ ] 위협 모델과 개인정보 점검
- [ ] 운영·롤백 절차
