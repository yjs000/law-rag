# 실행 계획 0007: Production 자연어 검색과 단계별 관측

상태: 구현 완료, Production 적용 대기
작성일: 2026-07-18
소유자: Codex

## 목적과 사용자 결과

AI가 비활성화된 검색 전용 모드에서도 자연스러운 한국어 질문이 관련 법령 근거를 반환하게 한다. 로그인 사용자의 질문 이력에는 입력 검증, 조문 파싱, 임베딩, 검색 전략과 후보 수, 생성·근거 게이트, 최종 결과를 구조화해 저장하여 답변 실패 원인을 재현하고 RAG 개선 근거로 사용할 수 있게 한다.

## 범위와 비범위

범위는 공유 질의 정규화, PostgreSQL의 제목·표제·본문 검색, 엄격 검색 후 완화 검색, 검색 전용 독립 동작, 질문 진단 JSONB, NFTC 기술기준 대형 청크 분할, 실제 코퍼스에서 검증한 첫 질문 예시와 회귀 테스트다.

OpenAI Production 키 연결, 실제 Terra 품질 평가, 법률 해석 정답 확정, 국가법령정보 Open API 외 출처 추가는 제외한다.

## 측정 가능한 완료 조건

- `확인할`, `알려주세요` 같은 질문 표현을 제거한 정규화 결과가 검색에 사용된다.
- 엄격 다단어 검색이 0건이면 안전하게 생성한 OR 질의로 완화 검색하며 제목·표제·본문을 모두 검색한다.
- 임베딩이 없거나 AI가 꺼져도 대표 자연어 질문이 키워드 근거를 반환한다.
- 직접 조문 질의는 기존 정확 경로 검색을 유지하고, 존재하지 않는 경로는 진단에 남는다.
- 로그인 질문 이력에 파싱·임베딩·검색·생성·결과 단계가 JSONB로 저장된다.
- NFTC 형식의 기술기준 본문은 내부 인용 숫자를 조문으로 오인하지 않고 안정적인 기준 단위로 분할된다.
- 첫 화면 예시 질문은 실제 검색 계약 테스트를 통과한 자연어 질문만 사용한다.
- 전체 테스트, lint, typecheck, 문서 검사와 Production build가 통과한다.

## 검증 및 롤백

DB migration은 기존 `hybrid_search` 시그니처를 유지하고 인덱스와 함수 본문만 교체한다. 질문 진단 컬럼은 기본 빈 JSON 객체를 사용하여 기존 이력과 호환한다. 검색 회귀가 발생하면 새 정규화·완화 검색을 되돌리되 직접 경로 검색과 기존 content 인덱스는 유지한다.

## 결정 로그

- 2026-07-18: PGroonga 공식 문서의 다중 컬럼 가중 검색과 `OR` 질의 문법을 사용한다. 사용자 원문을 query syntax로 직접 전달하지 않고 한글·영문·숫자 토큰으로 안전하게 재구성한다.
- 2026-07-18: AI 연결과 키워드 검색을 분리하여 검색 전용 모드는 임베딩 없이도 동작해야 한다.
- 2026-07-18: 사용자 질문 원문은 기존 인증 이력에만 보관하고 운영 로그에는 추가하지 않는다. 진단은 단계·상태·개수·안전한 정규화 결과만 저장한다.
