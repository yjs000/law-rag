# 신규 사용자 온보딩

상태: 가정 기반 초안  
작성일: 2026-07-13

## 사용자와 문제

초기 사용자는 법률 자료를 찾고 근거를 빠르게 확인하려는 한국어 사용자로 가정한다. 사용자는 서비스가 법률 자문인지, 어떤 자료를 대상으로 하는지, 답변을 얼마나 신뢰해야 하는지 알기 어렵다.

## 사용자 결과

사용자는 첫 질문 전에 서비스의 범위와 한계를 이해하고, 첫 답변에서 인용 원문을 열어 확인할 수 있다.

## 흐름

1. 지원 자료, 기준일, 법률 자문 대체 불가 고지를 짧게 본다.
2. 예시 질문 또는 직접 질문으로 시작한다.
3. 사건 식별 가능 정보 입력을 피하라는 안내를 받는다.
4. 답변의 근거 표시 방법을 한 번 안내받는다.
5. 인용을 열어 원문 위치와 출처를 확인한다.
6. 질문 이력을 남기려면 Google 로그인 또는 계정 만들기 화면으로 이동하며, 익명 질문은 저장되지 않는다는 안내를 본다.
7. 익명 상태에서 첫 답변을 받은 뒤 `질문 기록을 남기려면 로그인하세요` 팝업에서 로그인하거나 닫고 계속 사용할 수 있다.

## 수용 기준

- 첫 화면에서 지원 관할·자료 범위·기준일을 찾을 수 있다.
- 고지는 질문을 방해하지 않지만 제출 전에 인지 가능하다.
- 예시 질문에는 실제 개인 사건정보가 포함되지 않는다.
- 첫 답변의 모든 인용은 유효한 원문 위치로 이동한다.
- 사용자는 온보딩 안내를 다시 열 수 있다.
- 로그인 유도 팝업은 첫 익명 질문의 응답이 성공한 뒤 세션당 한 번만 표시되고 질문 제출이나 답변 확인을 가리지 않는다.
- 팝업을 닫거나 로그인하지 않아도 계속 질문할 수 있으며 로그인 전 익명 질문은 서버 질문 이력에 소급 저장되지 않는다.
- 로그인 화면에서 Google 단일 공급자, 1년 보관, 익명 질문 소급 저장 안 함을 확인할 수 있다.
- 회원가입 화면에서 가입 효과, 필수 이용약관·개인정보 동의, 기존 사용자 로그인 전환을 확인할 수 있다.

## 결정 및 미결정

- 결정: 검색과 질문은 익명으로 사용할 수 있지만 질문 이력 저장은 로그인 사용자에게만 제공한다.
- 결정: Google 로그인·회원가입은 Supabase Auth PKCE로 연결하며 공개 환경변수 입력과 실제 계정 E2E는 실행 계획 0004에서 완료한다.
- 결정: 실제 서비스 로그인은 Google만 지원한다.
- 결정: 질문 이력은 생성일부터 1년 보존하며, 계정 삭제 시 사용자 관련 데이터를 모두 삭제한다.
- 결정: 가입 시 현재 서비스 이용약관과 개인정보 처리방침의 버전 및 동의 시각을 저장한다.
- 결정: OpenAI 활성화 후 로그인 계정의 베타 AI 한도는 하루 10회, 검색 전용은 하루 100회로 시작한다. 활성화 전에는 검색 전용만 제공한다.
- 미결정: 일반인/전문가별 별도 온보딩
