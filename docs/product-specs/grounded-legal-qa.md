# 에너지 사업 법령 채팅

## 사용자와 문제

초기 사용자는 에너지 사업을 검토하는 일반 사용자다. 사용자는 법령 이름보다 “지금 이 사업 단계에서 무엇을 허가·신고·점검해야 하는가”를 알고 싶다. 서비스는 법률 자문이나 승패 예측 대신 조사 출발점과 원문 경로를 제공한다.

## 핵심 여정

1. 어두운 테마 우선의 채팅 화면에서 기준일을 선택하고 자연어로 질문한다.
2. Terra 근거 답변 또는 검색 전용 모드를 선택한다. 다른 생성 모델은 지원하지 않는다.
3. 서비스가 허용 코퍼스에서 기준일 유효 조문을 검색한다.
4. 답변의 의무·예외·적용 조건 옆에서 인용 ID를 확인하고 같은 대화 흐름에서 원문을 펼친다.
5. AI가 없거나 근거가 부족하면 생성 결론 대신 검색 결과와 한계를 본다.
6. Google 계정으로 로그인한 사용자는 왼쪽 질문 목록에서 자신의 이력을 다시 열 수 있고, 익명 사용자의 질문은 저장되지 않는다.
7. 사업 단계 체크리스트를 Markdown을 기본으로, CSV 또는 PDF를 선택해 내보낸다.
8. 익명 사용자가 첫 질문의 답변을 받은 뒤 대화를 가리지 않는 로그인 안내에서 Google 목업 로그인으로 이동할 수 있다.
9. 로그인 사용자는 계정 대시보드에서 로그인 공급자, 보존 기간, 허용 모델, 장애 시 동작과 베타 계정 한도를 확인한다.
10. 검색 결과가 없으면 해당 사실과 원인을 명시하고 법령명·조문 번호·사업 쟁점을 보완해 다시 질문하도록 안내한다.
11. Terra가 현재 사용 불가이거나 Terra 요청이 검색 전용으로 폴백되면 전환 사실을 알리고 선택값도 실제 응답 모드와 일치시킨다.
12. 질문을 전송하면 작성기에서 전송 문구를 즉시 비우고, 답변을 기다리는 동안에도 다음 질문 초안을 작성할 수 있다.
13. 같은 대화에서 질문과 답변을 순서대로 이어 보고, 첫 질문을 제목으로 한 대화를 질문 기록에서 다시 연다.
14. 대화의 예상 모델 입력이 24,576 토큰 예산을 넘으면 다음 질문은 새 대화에서 시작하고 컨텍스트 초기화 안내를 본다.
15. 생성 중지 버튼으로 현재 응답 대기를 멈출 수 있으며, 로그인·대화 목록은 필요한 페이지와 상세만 비동기로 불러온다.

## MVP 허용 목록

전기사업법과 시행령·시행규칙, 분산에너지 활성화 특별법과 시행령, 신에너지 및 재생에너지 개발ㆍ이용ㆍ보급 촉진법, 전기안전관리법, NFPC 607, NFTC 607의 정확 명칭 9개다.

## 완료 기준

- 채팅형 질문·기준일 입력과 답변 안의 접이식 원문
- 데스크톱 왼쪽 질문 목록과 모바일 사이드바
- Terra·검색 전용 요청 선택, 다른 생성 모델 미지원 표시
- 주장별 존재하는 인용 ID와 정확 원문 반환
- 검색 전용 답변에서 근거별 문서명·조문 경로·표제·원문과 사실관계 대조 확인 항목 제공
- Terra 초안의 요약·주장·체크리스트가 인용 근거에 없는 규범 유형·숫자·부정 방향·과잉 일반화를 포함하면 검색 전용으로 안전하게 폴백
- 코퍼스 상태와 검색 전용 모드 표시
- 검색 결과 없음의 원인 표시와 질문 구체화 동작
- 기준일 유효 버전 검색
- `1조2항`, `제12조의3 제2항`처럼 입력한 조문 경로를 기준일 유효 원문 경로로 직접 검색
- AI 쿼터 오류의 안전한 폴백
- Terra 사용 불가 표시, 접근 가능한 전환 알림, 실제 응답 모드와 선택값 동기화
- 법률 자문 대체 불가 고지
- 목업 로그인과 로그인 사용자 전용 질문 이력
- 첫 질문 제목의 대화 이력, cursor 기반 요약 목록과 선택 대화 상세 지연 로딩
- 입력 토큰 예산 초과 시 자동 새 대화 전환과 컨텍스트 초기화 안내
- 진행 중 응답 대기 중지와 늦은 응답 무시
- 첫 익명 질문 응답 후 세션당 한 번만 표시되는 비차단 로그인 안내
- 로그인 계정·보존·모델 정책을 표시하는 계정 대시보드
- 질문 이력의 1년 만료와 계정 삭제 시 사용자 관련 데이터 전체 삭제
- 동일한 체크리스트 데이터에서 Markdown·CSV·PDF 생성
- 표지·브랜딩 없는 단순 PDF 출력본

## 비목표

판례·블로그·다른 법률 사이트 검색, 개인 사건 판단, 법적 승패 예측, PDF 기본 청킹, 모델 기억으로 누락 근거 보완은 지원하지 않는다.
