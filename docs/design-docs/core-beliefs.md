# 핵심 신념

상태: 승인 전 초안  
작성일: 2026-07-13

1. **인용은 장식이 아니라 제품 계약이다.** 사용자는 답변의 각 핵심 주장을 원문에서 확인할 수 있어야 한다.
2. **검색 실패를 생성으로 감추지 않는다.** 근거가 없거나 충돌하면 답변의 범위와 확신을 낮춘다.
3. **법률의 시간성을 일급 데이터로 다룬다.** 시행일, 개정 이력, 판결일, 기준일을 잃지 않는다.
4. **원문 추적성을 보존한다.** 모든 파생 데이터는 출처, 원본 버전, 변환 버전으로 돌아갈 수 있어야 한다.
5. **검색과 생성을 별도로 평가한다.** 좋은 문장 때문에 나쁜 검색이 숨겨지지 않아야 한다.
6. **정확한 좁은 범위가 모호한 넓은 범위보다 낫다.** MVP 범위를 명시하고 지원 밖 질문을 구분한다.
7. **모델은 교체 가능한 어댑터다.** 도메인 정책과 데이터 계약을 모델 프롬프트에만 가두지 않는다.
8. **경계에서 파싱한다.** 외부 API, 사용자 입력, 모델 출력은 검증 후 내부로 들인다.
9. **민감정보를 기본적으로 수집하지 않는다.** 필요성이 입증될 때만 최소 범위와 보존 기간을 정한다.
10. **품질은 자동화된 피드백 루프로 유지한다.** 평가셋, 구조 테스트, 문서 점검으로 드리프트를 조기에 발견한다.

## 제품 표현 원칙

- `법률 정보`와 `개별 사건에 대한 법률 자문`의 경계를 분명히 알린다.
- 답변에는 적용 기준일, 관할, 사용한 자료 유형을 표시한다.
- 확률을 근거처럼 제시하지 않는다. 불확실성은 이유와 다음 확인 방법으로 설명한다.
- 출처가 충돌하면 하나를 임의로 선택하지 않고 충돌을 보여준다.
