# 국가법령정보 Open API 수집 계약

## 범위

법률 근거는 `open.law.go.kr`이 제공하는 공동활용 Open API만 허용한다. `OC`는 `LAW_OPEN_API_OC` 비밀값이다. 화면 HTML 크롤링, PDF 청킹, 검색엔진과 다른 법률 사이트는 금지한다.

## 포맷 결정

JSON을 우선 요청하지만 채택 기준은 파싱 성공이 아니라 도메인 정규화 성공이다. 정확 명칭, 출처 ID, MST, 검색 가능한 조문이 없으면 XML로 한 번 폴백한다. 두 포맷 fixture는 핵심 필드와 조문 경로가 같아야 한다.

일시적 5xx, timeout, 네트워크 오류는 같은 포맷을 최대 3회 재시도한다. 재시도 소진은 수집 실패이며 포맷 폴백 조건이 아니다. 관측 URL에서 `OC`를 `[redacted]`로 치환한다.

## 대상과 순서

1. 법령: `lawSearch.do?target=eflaw&search=1&nw=3`
2. 행정규칙: `lawSearch.do?target=admrul&search=1&nw=1`
3. 허용 목록과 정확 명칭 결과가 정확히 1개인지 확인
4. `lawService.do?target=eflaw|admrul` 본문 조회
5. 조·항·호·목 구조 보존 및 원문 SHA-256 계산
6. 원문 Storage 저장, DB 정규화, 임베딩
7. 동일 문서/MST upsert로 멱등성 보장

버전 목록은 법령 `eflaw&nw=1`, 행정규칙 `admrul&nw=2`, 변경 탐지는 `lsHstInf`, 조문 변경 이력은 `lsJoHstInf`의 JSON/XML 응답만 사용한다. 최초 제정·시행 버전부터 현재까지 모든 MST와 조문 변경 이력을 수집하고 효력 기간을 계산한다. 같은 MST가 단계별 시행일을 여러 개 가질 수 있으므로 버전 키는 `MST + 시행일`로 보존한다. 응답 계약이 검증되지 않은 기능은 비활성화한다.

현행 본문은 `target=eflaw&ID`를 사용한다. 특정 과거 MST 본문은 연혁 목록의 상세링크 계약에 따라 `target=eflaw&MST`와 그 버전의 `efYd`를 함께 전달한다.

행정규칙 본문은 목록의 `행정규칙일련번호`를 `ID`로 전달한다. 안정적 `행정규칙ID`를 사용할 때만 `LID`로 전달한다.

행정규칙 JSON의 `AdmRulService.조문내용` 배열은 각 항목에서 `제n조` 표기를 추출해 안정적인 조문 경로로 만들고, 조 번호가 없는 항목은 `본문/단락n`으로 보존한다.

본문 JSON은 같은 조 번호를 `조문여부=전문`과 `조문여부=조문`으로 나누어 제공할 수 있다. 조문 본문 경로는 한 번만 저장하고 구조화 노드의 항·호·목은 계속 하위 경로로 파싱한다.
