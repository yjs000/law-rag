# 제품 감각

## 우리가 해결하려는 문제

법률 정보는 방대하고 구조가 복잡하며 시간에 따라 바뀐다. 검색 사용자는 관련 문서를 찾는 데 오래 걸리고, 생성형 답변은 그럴듯하지만 근거가 부정확할 수 있다. law-rag는 빠른 답변보다 검증 가능한 조사 출발점을 제공한다.

## 우선할 사용자 가치

1. 관련 근거를 놓치지 않음
2. 근거의 정확한 위치와 효력 시점 확인
3. 답변이 모르는 것을 분명히 표현
4. 원문 조사 시간을 줄이는 명확한 요약
5. 민감한 사건정보를 요구하지 않는 안전한 사용

## 판단 질문

- 사용자가 어떤 결정을 더 정확하거나 빠르게 할 수 있는가?
- 지원하는 법률 범위를 넓히는가, 기존 범위의 신뢰도를 높이는가?
- 오류가 발생했을 때 사용자가 알아차리고 원문으로 복구할 수 있는가?
- 모델 없이도 제품 가치의 일부가 남는가?
- 추가 데이터 수집이 사용자 가치에 비례하고 정당한가?

## 성공 신호(초안)

- 사용자가 답변 후 관련 인용을 열고 원문 조사로 이어감
- 전문가 표본 검토에서 중대 근거 오류가 감소함
- 같은 조사 업무의 완료 시간이 줄어듦
- 근거 부족 질문에서 사용자가 잘못된 단정 대신 범위를 이해함

## 피해야 할 대리지표

답변 길이, 대화 횟수, 생성 토큰, 클릭 수만으로 성공을 판단하지 않는다. 정확성·근거성·업무 완료와 함께 해석한다.
