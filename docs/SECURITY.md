# 보안 및 개인정보

## 보호 대상

- 사용자의 질문과 사건 관련 사실
- 계정·조직·권한 정보
- 데이터 제공처 및 모델 API 자격증명
- 원문과 파생 색인의 무결성
- 프롬프트, 평가셋, 운영 로그

## 주요 위협

- 질문 또는 수집 문서를 통한 프롬프트 주입
- 모델 출력의 허위 인용과 권한 밖 문서 참조
- 로그, 분석, 캐시를 통한 개인정보 누출
- 외부 데이터 변조 또는 오래된 문서의 최신 문서 위장
- 원문 렌더링 XSS, SSRF, 악성 링크
- 관리자 수집/삭제/재색인 기능의 권한 오용
- 모델 제공자에게 필요 이상의 민감정보 전송

## 기본 통제

- 사용자·외부 문서·모델 출력을 모두 신뢰하지 않는 입력으로 검증한다.
- 인증은 인증, 인가는 서버의 자원 단위 정책으로 분리한다.
- 비밀은 코드·Git·로그와 분리하고 정기 회전한다.
- 전송 및 저장 암호화, 최소 권한, 운영자 작업 감사를 적용한다.
- 원문 URL은 허용된 출처와 프로토콜만 사용하고 서버 요청을 제한한다.
- 모델에는 필요한 최소 컨텍스트만 보내고 제공자의 보존/학습 설정을 검토한다.
- 삭제·보존·정정이 원문, 청크, 색인, 캐시, 백업에 전파되도록 설계한다.

## 개인정보 원칙

- 이름, 주민등록번호, 연락처, 주소, 사건번호 입력을 기본적으로 요구하지 않는다.
- 제품에 꼭 필요한 필드와 보존 기간을 데이터 목록에 기록한다.
- 분석 이벤트에는 원문 질문을 기본 포함하지 않는다.
- 실제 사용자 데이터를 평가셋으로 전환할 때 별도 적법 근거와 비식별 검토를 거친다.
- 익명 질문은 저장하지 않는다. 목업 단계에서는 로그인한 목업 사용자 질문만 로컬 목업 저장소에 저장하며 운영 데이터로 전환하지 않는다.
- 실제 인증은 Google OAuth만 지원하고 공급자가 발급한 식별자는 내부 사용자 ID와 분리해 저장한다.
- 로그인 질문 이력은 생성일부터 1년이 지나면 자동 삭제한다.
- 계정 삭제 시 질문·질문 이력·세션·내보내기·피드백 등 사용자 ID와 연결된 모든 원본·파생 데이터를 삭제한다. 백업 사본은 복구 보존주기가 끝날 때 제거하고 그 기간에는 운영 복원 외 용도로 사용하지 않는다.
- 개인정보 처리방침, 국외 이전, 위탁, 보유 기간은 배포 관할의 법률 검토 후 확정한다.

## 취약점 보고

공개 서비스 전 전용 보안 연락 경로와 대응 SLA를 정한다. 실제 비밀이나 개인 데이터는 공개 이슈에 올리지 않는다.

## 출시 전 필수 검토

- 데이터 흐름도와 위협 모델
- 의존성/비밀/정적 분석
- 권한 우회, XSS, SSRF, 프롬프트 주입, 인용 위조 테스트
- 백업·삭제·사고 대응 훈련
- 데이터 제공처 이용조건 및 개인정보 법률 검토
