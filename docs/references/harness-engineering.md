# Harness Engineering 적용 메모

출처: [OpenAI, 하네스 엔지니어링: 에이전트 우선 세계에서 Codex 활용하기](https://openai.com/ko-KR/index/harness-engineering/)  
확인일: 2026-07-13

## 이 저장소에 적용한 내용

- AGENTS.md를 긴 백과사전이 아닌 짧은 지도와 작업 계약으로 둔다.
- 세부 지식을 분류·색인된 docs/에 보관하고 점진적으로 읽게 한다.
- 아키텍처 경계와 의존성 방향을 초기에 명시한다.
- 큰 작업은 진행·결정 로그가 포함된 버전 관리 실행 계획으로 다룬다.
- 품질, 보안, 신뢰성, 제품 감각을 독립 문서로 둔다.
- 장기적으로 문서 링크, 신선도, 구조 규칙을 CI에서 검사한다.

## 아직 적용하지 않은 내용

- 문서 린터와 정기 문서 정원 관리 자동화
- 구조적 아키텍처 테스트와 맞춤 오류 메시지
- 격리된 워크트리별 앱/관측 가능성 스택
- 자동 평가·리뷰·복구 루프

제품·기술 스택이 결정된 뒤 실행 계획과 기술 부채 추적기에 반영한다.
