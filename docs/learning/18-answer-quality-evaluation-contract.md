# 질문·근거·답변 평가 계약

작성일: 2026-07-15

## 왜 예시 질문만으로는 부족한가

“전기사업 허가가 필요한가?”라는 질문 목록만 있으면 검색기가 어떤 문서와 조문을 찾아야 하는지, 답변이 어디까지 말해도 되는지 판단할 수 없다. 반대로 기대 문서만 지정하면 맞는 문서를 찾고도 근거에 없는 면제·자동 승인·안전 보장을 덧붙이는 답변을 놓칠 수 있다.

이번 평가 시드는 질문과 함께 다음 항목을 하나의 회귀 계약으로 고정한다.

1. 허용 목록의 기대 문서명
2. 기대 조문 경로
3. 국가법령정보 공동활용 Open API 원문을 나타내는 명시적 목업 `SearchHit`
4. 사람이 읽을 수 있는 참고 답변
5. 답변에 반드시 남아야 할 핵심 용어
6. 근거를 넘어서는 금지 결론
7. 검색 결과 상태와 근거 없음 사유

데이터는 `apps/api/tests/fixtures/answer_quality_cases.json`, 실행 계약은 `apps/api/tests/test_answer_quality_contract.py`에 있다.

## 다섯 사례가 확인하는 것

- **허가**: 전기사업, 허가 주체, 허가 의무가 함께 남아야 한다.
- **신고/변경**: 중요사항 변경허가 원문을 근거로 “모든 변경은 신고만 하면 된다”고 결론 내리면 안 된다.
- **전기저장시설 안전**: NFPC 607의 목적과 적용범위를 “화재가 절대 발생하지 않는다”는 보장으로 바꾸면 안 된다.
- **직접 조문 경로**: `전기사업법 제7조 제1항` 질문이 `제7조/항①` 경로와 원문으로 왕복해야 한다.
- **범위 밖/근거 없음**: 판례에 의한 개인 형사사건 승패 예측에는 인용을 만들지 않고 `no_matching_evidence`를 반환해야 한다.

## 결정적 지표의 의미

테스트는 허용 목록 준수, 기대 문서·경로 집합, 인용문과 목업 원문의 정확 일치, 필수 용어 포함, 금지 결론 부재, 결과 상태를 검사한다. 이 계약에서는 각 긍정 검사가 100%이고 금지 결론은 0건이어야 한다. 하나라도 어기면 회귀다.

이 지표는 법률 의미를 완전히 판정하는 점수가 아니다. 특히 문구 포함 검사는 동의어, 부정 범위, 조건과 예외의 복잡한 관계를 이해하지 못한다. 참고 답변도 유일한 정답이 아니라 현재 목업 근거에서 허용되는 최소 의미를 설명한다.

## 직접 실행

저장소의 `apps/api`에서 실행한다.

```powershell
..\..\.venv\Scripts\python.exe -m pytest tests/test_answer_quality_contract.py -q
..\..\.venv\Scripts\python.exe -m ruff check tests/test_answer_quality_contract.py
```

현재 결정적 계약은 11개 테스트로 구성된다.

## 아직 증명하지 않은 것

- Production 또는 로컬 실제 코퍼스에서 기대 조문을 검색하는지
- 실제 Terra 출력이 다양한 표현과 복합 조건에서도 법률적으로 정확한지
- Recall@k, MRR/nDCG, citation precision/coverage의 실제 베이스라인
- 시행일 변경, 여러 문서 결합, 상충 근거를 올바르게 처리하는지
- 법률 전문가 블라인드 검토에서 중대 오류가 없는지

다음 단계는 실제 코퍼스의 버전을 고정한 회귀셋을 실행하고, 모델·프롬프트 버전과 함께 결과를 기록하는 것이다. 그 뒤 별도 블라인드 셋과 법률 전문가 표본 검토로 결정적 게이트의 오탐·미탐을 측정해야 한다. 실제 사용자 질문은 적법 근거와 비식별 검토 없이 평가셋에 넣지 않는다.
