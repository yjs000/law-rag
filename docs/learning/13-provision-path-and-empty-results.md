# 조문 경로 검색과 빈 결과 계약

작성일: 2026-07-15

## 문제

`1조2항은?`은 조문 본문에 그대로 들어 있는 키워드가 아니다. 저장소에는 조문 위치가 별도 `path` 컬럼으로 저장되므로 기존 본문 중심 검색은 결과를 반환하지 못했다. 또한 빈 결과 응답은 한계 문구만 포함해 클라이언트가 일반 검색 불일치와 조문 경로 부재를 구별하기 어려웠다.

## 구현

- 사용자 표기 `1조2항`을 정규 경로 `제1조/항2`로 해석한다.
- 현재 숫자 경로와 수집 원문의 기호 경로(`항②`, `호2.`, `목가.`)를 모두 정확 조회한다.
- 법령명이 없으면 같은 경로를 가진 기준일 유효 법령을 제목순으로 반환한다. 법령명이 있으면 그 법령만 조회한다.
- 경로 질의는 경로 결과만 반환해 제목이나 숫자가 우연히 포함된 다른 조문이 섞이지 않게 한다.
- 질문 응답은 `result_status=no_results`와 `requested_path_not_found` 또는 `no_matching_evidence`를 반환하며, 요약과 한계에도 결과 없음과 원인을 명시한다.

## 안전 경계

조문 번호만으로 특정 법령을 추정하지 않는다. 기준일 유효성 필터와 MVP 정확 명칭 허용 목록은 기존 검색과 동일하게 유지한다. 원문에 존재하지 않는 조문을 생성하거나 모델 기억으로 보완하지 않는다.

## 검증

- 조·가지조문·항·호·목 정규화 경계
- 법령명 없는 동일 경로의 다중 법령 반환
- 정확 법령명 지정 시 해당 법령 한정
- 시행 전 버전 제외
- 없는 경로의 API 상태·원인 코드·사용자 문구

실제 Supabase에서는 `3조2항`이 세 법령의 `제3조/항②`를 반환하고, 법령명을 붙이면 한 법령으로 좁혀지는 것을 확인했다. `1조2항`은 현재 9개 MVP 법령 어디에도 해당 경로가 없어 정상적인 빈 결과다. 이 검증 중 법령명 없는 질의의 nullable SQL 파라미터가 asyncpg에서 모호해지는 문제도 발견해 `text` 타입을 명시했다.
