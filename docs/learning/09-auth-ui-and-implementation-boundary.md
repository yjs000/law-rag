# 09 인증 UI와 구현 경계

작성일: 2026-07-15

## 이번 마일스톤

Google 단일 공급자 정책 안에서 로그인과 신규 계정 화면을 분리했다. 로그인 화면은 질문 기록 보존과 익명 질문 비소급 저장을 설명한다. 회원가입 화면은 질문 이력·데이터 삭제 효과와 필수 약관·개인정보 동의를 먼저 보여준다.

이번 변경은 UI만 다룬다. Google 버튼은 OAuth나 목업 API를 호출하지 않고 실제 연결이 남아 있음을 인라인 상태로 알린다. 인증 백엔드·Supabase·세션·RLS 작업은 실행 계획 `0004-google-authentication.md`의 TODO다.

## 왜 로그인과 가입을 나누나

Google OAuth 요청 자체는 같아도 사용자의 의도와 필요한 안내는 다르다. 기존 사용자는 기록 복원을 기대하고, 신규 사용자는 어떤 데이터가 저장되고 언제 삭제되는지 동의 전에 알아야 한다. 화면을 나누되 공급자와 자격 증명 방식은 늘리지 않았다.

## 긴 문자열 경계

법률 질문에는 URL, 식별자, 붙여넣은 코드처럼 공백 없는 문자열이 들어올 수 있다. 사용자 말풍선과 질문 작성기에 `overflow-wrap: anywhere`, `word-break: break-word`, 가로 overflow 차단을 적용해 콘텐츠가 중앙 대화 폭을 넓히지 않게 했다.
