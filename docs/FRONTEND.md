# 프론트엔드 아키텍처

상태: Next.js 16·React 19 구현 중

## 책임

- 질문 작성과 필터 입력
- 스트리밍 또는 완료형 답변 표시
- 주장과 인용의 양방향 탐색
- 원문 위치의 안전한 렌더링
- 로딩, 빈 결과, 부분 실패, 재시도 상태
- 접근성, 국제화, 최소 분석 이벤트
- 로그인 사용자의 대화 목록과 계정·모델 정책 대시보드
- 요청별 Terra/검색 전용 선택

## 화면 구조

- 데스크톱은 대화 이력 사이드바, 모델 선택 헤더, 중앙 대화, 하단 질문 작성기로 구성한다.
- 모바일에서는 사이드바를 기본으로 닫고 명시적 메뉴와 스크림으로 연다.
- 첫 화면은 제품 목적·출처 범위·예시 질문을 설명하며, 질문 뒤에는 사용자 메시지와 검증된 답변을 같은 흐름에 쌓는다.
- 원문은 별도 고정 열 대신 답변 안의 접이식 항목으로 제공한다. 인용 버튼은 해당 원문을 열고 스크롤한다.
- 기본 색상 체계는 dark이며 WCAG 2.2 AA 대비와 키보드 포커스를 유지한다.

## 경계

- 서버 응답은 런타임 스키마로 검증한 뒤 UI 상태로 변환한다.
- 서버의 도메인 모델을 화면 전역 상태에 그대로 복제하지 않는다.
- 모델 제공자나 데이터 저장소를 브라우저에서 직접 호출하지 않는다.
- 법률 원문은 HTML로 신뢰하지 않고 안전한 텍스트/허용 마크업으로 렌더링한다.
- 인증 토큰과 민감 질문을 영구 브라우저 저장소에 기본 저장하지 않는다.

## 상태 모델

각 질의는 idle → validating → retrieving → answering → validating_answer → success를 기본 흐름으로 갖고, 모든 단계에서 명시적 partial 또는 error 상태를 허용한다. 네트워크 재시도는 중복 질의를 만들지 않도록 요청 ID를 사용한다.

## 테스트

- 표현 컴포넌트 단위 테스트
- API 계약/상태 전이 통합 테스트
- 질문 → 답변 → 인용 → 원문 핵심 종단 테스트
- 키보드, 스크린리더 의미, 색상 대비 자동 검사
- 원문/모델 출력에 대한 XSS 회귀 테스트

구현은 `apps/web/app/page.tsx`의 클라이언트 상태와 `apps/web/lib/api-client.ts`의 같은 출처 API 경계로 분리한다. 대표 검증 명령은 `pnpm lint:web`, `pnpm typecheck`, `pnpm test:web`, `pnpm build:web`이다.

- 2026-07-14: 실제 선택 가능한 응답은 Terra와 검색 전용만 둔다. 다른 생성 모델은 품질 검증 전 활성화하지 않는다.
