# 프론트엔드 아키텍처

상태: 기술 스택 미결정

## 책임

- 질문 작성과 필터 입력
- 스트리밍 또는 완료형 답변 표시
- 주장과 인용의 양방향 탐색
- 원문 위치의 안전한 렌더링
- 로딩, 빈 결과, 부분 실패, 재시도 상태
- 접근성, 국제화, 최소 분석 이벤트

## 경계

- 서버 응답은 런타임 스키마로 검증한 뒤 UI 상태로 변환한다.
- 서버의 도메인 모델을 화면 전역 상태에 그대로 복제하지 않는다.
- 모델 제공자나 데이터 저장소를 브라우저에서 직접 호출하지 않는다.
- 법률 원문은 HTML로 신뢰하지 않고 안전한 텍스트/허용 마크업으로 렌더링한다.
- 인증 토큰과 민감 질문을 영구 브라우저 저장소에 기본 저장하지 않는다.

## 상태 모델

각 질의는 idle → validating → retrieving → answering → validating_answer → success를 기본 흐름으로 갖고, 모든 단계에서 명시적 partial 또는 error 상태를 허용한다. 네트워크 재시도는 중복 질의를 만들지 않도록 요청 ID를 사용한다.

## 테스트

- 표현 컴포넌트 단위 테스트
- API 계약/상태 전이 통합 테스트
- 질문 → 답변 → 인용 → 원문 핵심 종단 테스트
- 키보드, 스크린리더 의미, 색상 대비 자동 검사
- 원문/모델 출력에 대한 XSS 회귀 테스트

구체적인 디렉터리, 명령, 렌더링 방식은 프레임워크 ADR 이후 갱신한다.
