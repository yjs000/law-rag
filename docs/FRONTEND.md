# 프론트엔드 아키텍처

상태: Next.js 16·React 19 구현 중

## 책임

- 질문 작성과 필터 입력
- 스트리밍 또는 완료형 답변 표시
- 주장과 인용의 양방향 탐색
- 원문 위치의 안전한 렌더링
- 로딩, 빈 결과, 부분 실패, 재시도 상태
- 접근성, 국제화, 최소 분석 이벤트
- 로그인 사용자의 대화 목록과 계정·모델 정책 대시보드
- 요청별 Terra/검색 전용 선택
- 검색 결과 없음의 원인과 질문 구체화 안내

## 화면 구조

- 데스크톱은 대화 이력 사이드바, 모델 선택 헤더, 중앙 대화, 하단 질문 작성기로 구성한다.
- 모바일에서는 사이드바를 기본으로 닫고 명시적 메뉴와 스크림으로 연다.
- 첫 화면은 제품 목적·출처 범위·예시 질문을 설명하며, 질문 뒤에는 사용자 메시지와 검증된 답변을 같은 흐름에 쌓는다.
- 원문은 별도 고정 열 대신 답변 안의 접이식 항목으로 제공한다. 인용 버튼은 해당 원문을 열고 스크롤한다.
- 기본 색상 체계는 dark이며 WCAG 2.2 AA 대비와 키보드 포커스를 유지한다.

## 경계

- 서버 응답은 런타임 스키마로 검증한 뒤 UI 상태로 변환한다.
- 서버의 도메인 모델을 화면 전역 상태에 그대로 복제하지 않는다.
- 모델 제공자나 데이터 저장소를 브라우저에서 직접 호출하지 않는다.
- 법률 원문은 HTML로 신뢰하지 않고 안전한 텍스트/허용 마크업으로 렌더링한다.
- 인증 토큰과 민감 질문을 영구 브라우저 저장소에 기본 저장하지 않는다.

## 상태 모델

각 질의는 idle → validating → retrieving → answering → validating_answer → success를 기본 흐름으로 갖고, 모든 단계에서 명시적 partial 또는 error 상태를 허용한다. 네트워크 재시도는 중복 질의를 만들지 않도록 요청 ID를 사용한다.

질문 제출 시 전송 문구는 대화 메시지 상태로 복사하고 작성기 초안은 즉시 비운 뒤 포커스를 작성기로 돌려준다. 진행 중 요청의 중복 제출 버튼은 비활성화하지만 작성기 자체는 활성 상태로 유지해 사용자가 다음 질문을 미리 작성할 수 있게 한다. 응답 완료와 실패는 사용자가 진행 중에 작성한 새 초안을 변경하지 않는다.

`result_status=no_results`는 요청 실패가 아니라 정상적으로 검색을 마쳤지만 표시할 근거가 없는 성공 상태다. UI는 `no_results_reason` 코드를 사용자 문장으로 변환하고, 구버전 응답은 `sections`와 `citations`가 모두 빈 경우 같은 상태로 처리한다. 빈 결과에서는 로그인 유도보다 법령명·조문 번호·사업 쟁점을 보완하는 재질문 안내를 우선한다.

응답 모드는 화면의 선택값이 아니라 API의 실제 `mode`를 기준으로 동기화한다. 기본 선택은 Terra다. `GET /v1/corpus/status`가 `ai_available=false`이면 Terra를 `현재 사용 불가`로 비활성화하고, 접근 가능한 상태 알림으로 `Terra 한도 초과로 검색 전용으로 전환합니다.`를 표시한 뒤 검색 전용을 선택한다. Terra 요청의 응답이 `mode=search_only`이면 `requested_answer_mode`와 `fallback_reason`을 사용해 같은 전환을 수행한다. 새 필드가 없는 구버전 API도 요청값과 실제 `mode`의 차이로 폴백을 판별한다.

## 테스트

- 표현 컴포넌트 단위 테스트
- API 계약/상태 전이 통합 테스트
- 질문 → 답변 → 인용 → 원문 핵심 종단 테스트
- 키보드, 스크린리더 의미, 색상 대비 자동 검사
- 원문/모델 출력에 대한 XSS 회귀 테스트

구현은 `apps/web/app/page.tsx`의 클라이언트 상태와 `apps/web/lib/api-client.ts`의 같은 출처 API 경계로 분리한다. 대표 검증 명령은 `pnpm lint:web`, `pnpm typecheck`, `pnpm test:web`, `pnpm build:web`이다.

## 결정 기록

- 2026-07-14: 첫 화면과 질문 흐름을 3열 워크벤치에서 ChatGPT형 단일 대화 흐름으로 변경했다. 작은 화면에서 답변과 원문을 순서대로 읽기 쉽고 이력 복원이 명확하기 때문이다.
- 2026-07-14: 실제 선택 가능한 응답은 Terra와 검색 전용만 둔다. 다른 생성 모델은 품질 검증 전 활성화하지 않는다.
- 2026-07-14: 별도 비밀번호 회원가입 폼을 만들지 않고 Google 로그인 첫 진입을 가입과 로그인에 함께 사용한다.
- 2026-07-15: Google 공급자는 하나로 유지하되 로그인과 회원가입 UI를 분리한다. 신규 사용자는 데이터 보관·삭제 효과와 필수 동의를 가입 전에 확인해야 하기 때문이다.
- 2026-07-15: 실제 OAuth 연결 전 Google 버튼은 목업 로그인 대신 구현 예정 상태를 표시한다. UI 완료와 인증 보안 완료를 혼동하지 않기 위함이다.
- 2026-07-15: 빈 검색 결과를 일반 검색 전용 응답과 구분하고 원인·재질문 방법을 함께 표시한다. 빈 결과 직후에는 질문 해결과 무관한 로그인 유도를 질문 구체화 안내로 바꾼다.
- 2026-07-15: Terra의 가용성과 실제 응답 모드를 선택 UI에 동기화하고, 자동 검색 전용 전환은 상태 알림으로 사용자에게 명시한다.
- 2026-07-15: 제출 직후 작성기 초안을 비우고 요청 처리 중에도 다음 질문 입력을 허용한다. 진행 중 요청은 한 번에 하나만 전송해 현재 단일 응답 상태와 충돌하지 않게 한다.
