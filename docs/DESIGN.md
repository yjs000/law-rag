# 제품 디자인 원칙

## 목표

법률 정보를 위압적이지 않게 설명하되, 단순화 과정에서 출처와 불확실성을 숨기지 않는다.

## 원칙

- 답변보다 근거 확인 경로를 먼저 설계한다.
- 출처, 기준일, 관할, 문서 유형을 일관된 시각 계층으로 표시한다.
- 확신을 색상 하나로 표현하지 않고 텍스트 이유를 함께 제공한다.
- 경고를 남발하지 않는다. 행동이 필요한 위험만 구체적으로 안내한다.
- 전문 용어에는 원문 용어를 보존하면서 쉬운 설명을 제공한다.
- 인용을 열어도 질문과 답변 맥락을 잃지 않게 한다.

## 핵심 화면(초안)

1. 검색/질문 시작 화면
2. 답변과 주장별 인용 패널
3. 원문 뷰어와 해당 위치 강조
4. 검색 필터 및 범위 설명
5. 오류·근거 부족·충돌 상태
6. 피드백 제출 흐름

## 접근성

- 키보드만으로 질문, 인용 탐색, 원문 닫기가 가능해야 한다.
- 의미 있는 제목 구조, 포커스 표시, 스크린리더 레이블을 제공한다.
- 색상 대비는 WCAG 2.2 AA를 목표로 하며 색상만으로 상태를 구분하지 않는다.
- 긴 법률 텍스트의 줄 길이, 확대, 문단 이동을 지원한다.

시각 토큰과 컴포넌트는 프론트엔드 기술 선택 후 별도 디자인 시스템 문서에서 확정한다.
