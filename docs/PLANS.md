# 실행 계획 운영법

## 실행 계획이 필요한 경우

- 여러 구성 요소나 마이그레이션을 건드리는 변경
- 하루 이상 또는 여러 작업 세션이 필요한 변경
- 데이터, 보안, 신뢰성, 외부 서비스 결정이 포함된 변경
- 실패 시 되돌림이나 단계적 배포가 필요한 변경

작고 명확한 수정은 이슈나 작업 설명만으로 충분하다.

## 위치

- 진행 중: docs/exec-plans/active/
- 완료: docs/exec-plans/completed/
- 알려진 장기 부채: docs/exec-plans/tech-debt-tracker.md

## 필수 섹션

1. 목적과 사용자 결과
2. 범위와 비범위
3. 측정 가능한 완료 조건
4. 순서 있는 단계와 체크리스트
5. 검증 및 롤백
6. 결정 로그
7. 날짜가 포함된 진행 기록
8. 미결정과 차단 요소

계획은 살아 있는 문서다. 작업 중 체크박스, 발견, 결정, 범위 변화를 갱신한다. 완료 후 실제 결과, 검증 증거, 남은 부채를 기록하고 completed/로 이동한다.
